**IMPORTANT:** The sidebar components live in SharedUI but need a `NavigationCategory`-like type to reference. Since `NavigationCategory` is defined in the App target (MainView.swift), the sidebar components must be generic or accept a protocol. The cleanest approach: define a `SidebarItem` protocol in SharedUI that `NavigationCategory` will conform to in the App target.

However, since SidebarView needs to know about sections (LIBRARY vs TOOLS), icon mappings, and the specific items — and SharedUI already depends on Core — the most pragmatic approach is:

**Option chosen:** Move `NavigationCategory` semantics into SharedUI as a concrete enum `SidebarDestination` that both SharedUI components and App target can use. This avoids protocol complexity and keeps the sidebar self-contained.

**SidebarSectionHeader.swift** — Section header component:
- `public struct SidebarSectionHeader: View`
- Takes `title: String` and `isCompact: Bool`
- When `isCompact`: renders a `Divider()` with `.padding(.vertical, Spacing.xs)`
- When expanded: renders `Text(title)` with `.font(.system(size: 11, weight: .medium))`, `.foregroundStyle(Ayu.fgSecondary)`, `.textCase(.uppercase)`, leading alignment, `.padding(.horizontal, Spacing.sm)`, `.padding(.top, Spacing.sm)`, `.padding(.bottom, Spacing.xxs)`

**SidebarItemView.swift** — Individual sidebar row with pill + hover:
- `public struct SidebarItemView: View`
- Properties:
  - `let title: String`
  - `let icon: NSImage` (Lucide icon passed in)
  - `let isSelected: Bool`
  - `let isCompact: Bool`
  - `let namespace: Namespace.ID`
  - `let action: () -> Void`
- `@State private var isHovered = false`
- Body: `Button(action:)` with `.buttonStyle(.plain)`
- Content layout:
  - `HStack(spacing: Spacing.sm)` containing:
    - Lucide icon: copy the NSImage, set `.isTemplate = true`, wrap in `Image(nsImage:)` with `.resizable().aspectRatio(contentMode: .fit).frame(width: 18, height: 18)`
    - If not compact: `Text(title).font(AppFont.body)` with `.transition(.opacity.combined(with: .move(edge: .leading)))`
  - `.frame(maxWidth: .infinity, alignment: isCompact ? .center : .leading)`
  - `.foregroundStyle(isSelected ? Ayu.accent : Ayu.fgPrimary)`
- Padding: `.padding(.horizontal, Spacing.sm)`, `.padding(.vertical, Spacing.xs)`
- Background (using `.background { }` closure):
  - If `isSelected`: `RoundedRectangle(cornerRadius: Radius.sm)` with `.fill(Ayu.accent.opacity(0.15))` and `.strokeBorder(Ayu.accent, lineWidth: 1)` and `.matchedGeometryEffect(id: "activeIndicator", in: namespace)`
  - Else if `isHovered`: `RoundedRectangle(cornerRadius: Radius.sm).fill(Ayu.bgTertiary)`
- `.contentShape(.rect)` for hit testing (macOS 15 scroll fix)
- `.onHover { hovering in withAnimation(Motion.curveFast) { isHovered = hovering } }`
- Compact mode active state: same pill but wrapping icon only (the frame centers the icon)
- Add `.help(title)` for tooltip in compact mode (only when `isCompact`)

**SidebarView.swift** — Full sidebar container:
- `public struct SidebarView<Destination: Hashable & CaseIterable & Identifiable>: View` — NO. Too complex. Use a concrete approach instead.
- Actually, define the sidebar to accept bindings and configuration externally. The simplest approach:

```swift
public struct SidebarView: View {
    public struct Item: Identifiable {
        public let id: String
        public let title: String
        public let icon: NSImage
        public let section: String // "LIBRARY" or "TOOLS"

        public init(id: String, title: String, icon: NSImage, section: String) { ... }
    }

    @Binding var selectedItemID: String?
    let items: [Item]
    @AppStorage("sidebarCompact") private var isCompact = false
    @Namespace private var sidebarNamespace
    let onSettingsTapped: () -> Void
}
```

- Body structure:
  1. **Toggle button** at top: `Button` with `Image(systemName: "sidebar.left")`, `.foregroundStyle(Ayu.fgSecondary)`, `.buttonStyle(.plain)`, wrapping `isCompact.toggle()` in `withAnimation(Motion.curveDefault)`. Padding: `.padding(.horizontal, Spacing.sm)`, `.padding(.vertical, Spacing.xs)`. Alignment: `.frame(maxWidth: .infinity, alignment: isCompact ? .center : .trailing)`.
  2. **ScrollView** with `VStack(spacing: Spacing.xxs)`:
     - Group items by section. For each unique section:
       - `SidebarSectionHeader(title: section, isCompact: isCompact)`
       - `ForEach(sectionItems)` rendering `SidebarItemView`
     - The action on each item: `withAnimation(Motion.curveSmooth) { selectedItemID = item.id }`
  3. **Spacer()** to push settings to bottom
  4. **Settings footer**: `Button` with `Image(systemName: "gearshape")` and optional `Text("Settings")` when not compact. `.foregroundStyle(Ayu.fgSecondary)`. `.buttonStyle(.plain)`. Same hover pattern. Calls `onSettingsTapped()`.
  5. `.padding(.horizontal, Spacing.xs)` on the ScrollView content
  6. **Background**: `.background(isCompact ? Color.clear : Ayu.bgSecondary)`
  7. **Column width**: DO NOT set `.navigationSplitViewColumnWidth` here — the consumer (MainView) sets it. The sidebar adapts to available width.

- All types/init must be `public`
- Add `@Environment(\.accessibilityReduceMotion) private var reduceMotion` — when reduce motion is true, use `.default` instead of `Motion.curveSmooth` for pill animation
