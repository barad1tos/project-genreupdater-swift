**MainView.swift — Sidebar replacement:**

5. Replace the existing `sidebar` computed property (which uses `List(selection:)`) with the new `SidebarView`:
   ```swift
   private var sidebar: some View {
       SidebarView(
           selectedItemID: Binding(
               get: { selectedCategory?.id },
               set: { newID in
                   selectedCategory = NavigationCategory.allInOrder.first { $0.id == newID }
               }
           ),
           items: NavigationCategory.allInOrder.map(\.sidebarItem),
           onSettingsTapped: {
               // Open Settings window via NSApp
               NSApp.sendAction(Selector(("showSettingsWindow:")), to: nil, from: nil)
           }
       )
       .navigationSplitViewColumnWidth(min: 160, ideal: 200, max: 260)
   }
   ```
   Note: the compact sidebar width is handled internally by SidebarView through its isCompact state — when compact, the content naturally shrinks. The `navigationSplitViewColumnWidth` applies to the expanded state. For compact mode to work correctly with NavigationSplitView, the SidebarView must communicate its desired width. The simplest approach: always use the expanded width range and let compact content center within the wider column. If this looks wrong, the executor can adjust to use a conditional `navigationSplitViewColumnWidth(min: isCompact ? 52 : 160, ...)` but this requires the isCompact state to be accessible from MainView. Consider adding `@AppStorage("sidebarCompact")` in MainView to read the same value.

   **Revised approach for compact width:** Add `@AppStorage("sidebarCompact") private var isSidebarCompact = false` in MainView. Use conditional width:
   ```swift
   .navigationSplitViewColumnWidth(
       min: isSidebarCompact ? 52 : 160,
       ideal: isSidebarCompact ? 52 : 200,
       max: isSidebarCompact ? 52 : 260
   )
   ```
