**CardLiftState.swift** -- Create the state machine types. All types must be `public` and `Sendable` (SPM boundary).

Types to create:
- `CardLiftPhase` enum: `.idle`, `.pressing`, `.lifted`, `.dismissing`. Conforms to `Equatable`, `Sendable`.
- `CardContentType` enum: `.artist(name: String)`, `.album(name: String, artistName: String)`. Conforms to `Equatable`, `Sendable`, `Hashable`.
- `CardLiftState` struct: properties `sourceID: String`, `contentType: CardContentType`, `phase: CardLiftPhase` (default `.pressing`), `sourceFrame: CGRect` (default `.zero`), `cascadeParent: CardLiftState?` (optional, for Artist -> Album drill-down). Conforms to `Equatable`, `Sendable`. The `cascadeParent` needs to be boxed since structs cannot have recursive stored properties -- use an indirect approach: store `parentSourceID: String?`, `parentContentType: CardContentType?`, `parentSourceFrame: CGRect?` as flat fields instead of recursive struct. This keeps `Sendable` conformance trivial.
- `CardGlowColor` enum with `public static func forContentType(_ type: CardContentType) -> Color` returning `Ayu.accent` for `.artist` and `Ayu.info` for `.album` per RESEARCH.md discretion recommendation.
- Add `import SwiftUI` for `Color` and `CGRect`.

**DoubleClickDetector.swift** -- NSViewRepresentable for macOS double-click detection.

Create `public struct DoubleClickDetector: NSViewRepresentable` following the pattern from RESEARCH.md:
- Takes `action: () -> Void` closure.
- Inner `public class DoubleClickNSView: NSView` that overrides `mouseDown(with:)` and fires action when `event.clickCount == 2`.
- `makeNSView` creates the view, `updateNSView` updates the action closure.
- `import AppKit` for NSView.
- The NSView must call `super.mouseDown(with: event)` to allow the event to propagate for single-click List selection.
- Add `// periphery:ignore` on the class if needed (NSViewRepresentable internals).

**NeonGlowBorder.swift** -- Breathing glow border with layered shadows.

Create `public struct NeonGlowBorder: View`:
- Parameters: `color: Color`, `cornerRadius: CGFloat`, `lineWidth: CGFloat` (default 1), `isActive: Bool` (default true).
- `@State private var glowOpacity: CGFloat = 0.4`
- `@Environment(\.accessibilityReduceMotion) private var reduceMotion`
- Body: `RoundedRectangle(cornerRadius: cornerRadius)` with:
  - `.stroke(color.opacity(glowOpacity), lineWidth: lineWidth)` -- hairline border per user decision
  - `.shadow(color: color.opacity(glowOpacity * 0.8), radius: 4)` -- inner glow
  - `.shadow(color: color.opacity(glowOpacity * 0.6), radius: 8)` -- mid glow
  - `.shadow(color: color.opacity(glowOpacity * 0.4), radius: 16)` -- outer glow
- `.onAppear`: if `isActive && !reduceMotion`, animate glowOpacity to 0.7 with `.easeInOut(duration: 2.5).repeatForever(autoreverses: true)`. If `reduceMotion`, set static `glowOpacity = 0.55`.
- Dark mode gets vivid glow, light mode gets muted glow: Use `@Environment(\.colorScheme)` and apply a multiplier -- dark: 1.0, light: 0.6 -- on all opacity values. This makes light-mode glow subtler per user decision.
- `.allowsHitTesting(false)` on the whole view so clicks pass through to card content.

**CardLiftOverlay.swift** -- Full overlay container.

Create `public struct CardLiftOverlay<Content: View>: View`:
- Parameters: `state: CardLiftState`, `containerSize: CGSize`, `onDismiss: () -> Void`, `@ViewBuilder content: () -> Content`.
- `@Environment(\.accessibilityReduceMotion) private var reduceMotion`
- Computed properties:
  - `cardWidth`: `min(containerSize.width * 0.7, 600)` per user decision (adaptive, max 600pt)
  - `isLifted`: `state.phase == .lifted`
  - `isPressing`: `state.phase == .pressing`
  - `scaleValue`: `.pressing` -> 0.97, `.lifted` -> 1.03, `.dismissing` -> 1.0, `.idle` -> 1.0
  - `cardOffset`: When `.lifted`, card should be vertically centered in container. When `.pressing`/`.dismissing`, offset derives from `state.sourceFrame` to animate from/to the row's position. Compute as: `.lifted` -> center Y offset, `.pressing`/`.idle`/`.dismissing` -> sourceFrame center Y relative to container center.

- Body (ZStack):
  1. **Dim overlay**: `Color.black.opacity(isLifted || isPressing ? 0.6 : 0)` with `.ignoresSafeArea()` and `.onTapGesture { onDismiss() }`. Dim timing synchronized with lift per user decision.
  2. **Card**: `content()` wrapped in:
     - `.frame(maxWidth: cardWidth)`
     - `.background(Ayu.bgPrimary, in: RoundedRectangle(cornerRadius: Radius.lg))`
     - `.clipShape(RoundedRectangle(cornerRadius: Radius.lg))`
     - `.overlay { NeonGlowBorder(color: CardGlowColor.forContentType(state.contentType), cornerRadius: Radius.lg) }`
     - `.ayuShadow(Shadow.floating)` per user decision (deep shadow elevation)
     - `.scaleEffect(scaleValue)`
     - `.offset(y: offsetForPhase)` -- animate position from source to center
  3. Apply animation: `.animation(reduceMotion ? nil : Motion.cardLiftSpring, value: state.phase)`

- Escape key handling: `.onKeyPress(.escape) { onDismiss(); return .handled }`
  Use `.onKeyPress` (macOS 14+) rather than `.onExitCommand` for more reliable behavior per RESEARCH.md pitfall 2.
- `.focusable()` to receive key events.

Content is visible immediately with the lift -- no delayed fade-in per user decision.
