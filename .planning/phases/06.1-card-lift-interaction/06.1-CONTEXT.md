# Phase 6.1: Card Lift Interaction - Context

**Gathered:** 2026-02-23
**Status:** Ready for planning

<domain>
## Phase Boundary

Build a Things 3-style Card Lift primitive in SharedUI (double-click triggers card lift with Ayu neon glow, background dims) and apply to Browse artist/album/track rows. The CardLift modifier is a reusable SharedUI ViewModifier. Other screens adopt in Phase 8.

</domain>

<decisions>
## Implementation Decisions

### Card content
- **Artist card**: Rich detail card — list of albums, genre statistics, mini-HeroGauge watermark. Full information view, not just an expanded row
- **Album card**: Tracklist + metadata — song list of the album, year, genre, track count. A compact version of BrowseDetailView, not the full panel
- **Card size**: Adaptive width — ~70% of content area width, maximum 600pt. Scales with window size
- **Card origin**: Animates FROM the row position upward to center — the row visually transforms into the card (Things 3 style)
- **Source row**: Stays visible in the list, dimmed together with the rest of the list at ~40% opacity. No gap left behind

### Animation
- **Lift curve**: Spring bounce with slight overshoot — feels like a physical object lifting off the surface
- **Scale**: Card scales up (~1.02-1.05x) AND gains deep shadow elevation — both effects combined
- **Pre-lift**: Press-in micro-animation (scale ~0.97) on double-click BEFORE the lift begins — button-press feel
- **Dismiss**: Reverse animation — card shrinks back toward the row position with spring physics
- **Dim timing**: Background dim animates synchronously with lift — one coordinated animation, not sequential
- **Content**: All card content visible immediately with the lift — no delayed fade-in

### Interaction model
- **Actions in card**: View only — card is informational, closing is the only interaction
- **Cascade navigation**: Double-click on an album row inside an artist card opens an album card — navigating deeper
- **Cascade behavior**: Previous card disappears (not stacked), new card appears. Escape returns to the previous card level
- **Maximum depth**: Artist → Album only (2 levels). Track-level cards are not needed — insufficient unique content
- **Dismissal**: Escape key or click-outside only — no visible X button. Clean Things 3-style appearance

### Glow and visual style
- **Glow behavior**: Gentle breathing pulse — opacity oscillates between 0.4 and 0.7, slow cycle
- **Glow color**: Different colors per content type — Artist uses accent, Album uses a different Ayu hue. Visually distinguishes hierarchy levels
- **Dark mode**: Vivid glow; Light mode: muted glow — adapts automatically via Ayu tokens
- **Corner radius**: Radius.lg (12pt) — consistent with existing design system components (StatCard, FilterChip)
- **Card background**: Ayu bgPrimary — same as content area, card distinguished by shadow + glow
- **Border**: Hairline border (1pt) in accent color underneath the glow — crisp edge definition plus soft glow aura
- **Dim overlay**: Color.black.opacity(0.6) — classic modal dim, neutral

### Claude's Discretion
- reduceMotion accessibility behavior (instant vs cross-fade fallback)
- Track card content layout (what metadata to show, since user deferred)
- Detail panel relationship (whether card replaces panel or overlays)
- Exact spring parameters (response, dampingFraction, blendDuration)
- Exact glow pulse timing (duration, easing)
- Specific Ayu colors for album glow differentiation

</decisions>

<specifics>
## Specific Ideas

- "Things 3-style" — the card lifts from the row's position, not a generic centered popover. The source row stays dimmed in place
- Cascade navigation feels like drill-down: Artist → double-click album → album card replaces artist card. Escape navigates back one level
- Press-in micro-animation before lift adds tactile feel — like pressing a physical button before it springs up
- Glow breathing effect should be subtle enough to not distract from content but vivid enough to signal the card is "alive"

</specifics>
