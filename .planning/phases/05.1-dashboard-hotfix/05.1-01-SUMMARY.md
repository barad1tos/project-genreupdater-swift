# Phase 5.1 Plan 01: Shimmer Timing Fix + Crossfade Stagger Summary

**Fixed first-launch empty-state flash with isLoadingTracks guard, added ZStack crossfade with 0.5s minimum shimmer hold and 50ms stagger cascade**

## Accomplishments
- Fixed core bug: `.task(id: tracks.count)` no longer transitions to `.emptyLibrary` while MusicKit is still loading
- ShimmerPlaceholder now has `.quickAction(height:)` shape case using `maxWidth: .infinity` instead of broken `CGFloat.infinity` fixed width
- Shimmer-to-content uses ZStack crossfade with `Motion.curveCrossfade` (0.5s easeInOut) and minimum 0.5s shimmer hold
- Stagger cascade: Hero gauge -> Metric cards -> Quick actions with 50ms delays on first load only
- Error states (permission denied, empty library, errors) appear instantly without crossfade
- `accessibilityReduceMotion` disables all stagger and crossfade animations
- 15s loading timeout shows error state with Retry button
- Shimmer gradient uses Ayu palette colors (bgSecondary.opacity(0.5))

## Decisions Made
- ZStack crossfade (not SwiftUI `.transition()`) to avoid layout jumps during shimmer-to-content swap
- `isLoadingTracks` guard in `refreshFromLive` is the core bug fix for first-launch empty-state flash
- Stagger only activates on first load (`isFirstLoad` flag); subsequent visits show all content immediately
- `accessibilityReduceMotion` environment value disables both crossfade and stagger animations
- `transitionToLive()` helper method consolidates state transition and `isFirstLoad` reset
