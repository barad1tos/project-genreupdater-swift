**1. Create `AppearanceMode.swift` in SharedUI/Theme/:**

Create a `public enum AppearanceMode: String, CaseIterable, Sendable` with three cases: `system`, `light`, `dark`.

Computed properties (all `public`):
- `colorScheme: ColorScheme?` — returns `nil` for system, `.light` for light, `.dark` for dark. Used by `preferredColorScheme(_:)`.
- `symbolName: String` — SF Symbol for the segmented picker: `"moon.fill"` (dark), `"circle.lefthalf.filled"` (system), `"sun.max.fill"` (light). Per user decision (locked).
- `accessibilityLabel: String` — VoiceOver labels: "System", "Light", "Dark".

Import `SwiftUI` (needed for `ColorScheme` type). File goes in `Packages/SharedUI/Sources/SharedUI/Theme/`.

**2. Wire appearance in `GenreUpdaterApp.swift`:**

Add `import AppKit` (needed for `NSApp.appearance` and `NSAppearance`).

Add `@AppStorage("appearanceMode") private var appearanceMode: AppearanceMode = .system` to `GenreUpdaterApp`.

On the `WindowGroup` content:
- Add `.preferredColorScheme(appearanceMode.colorScheme)` to `ContentView()`.
- Add `.onChange(of: appearanceMode) { _, newMode in applyAppKitAppearance(newMode) }` to apply AppKit layer.
- In the existing `.task { }`, add `applyAppKitAppearance(appearanceMode)` after `dependencies.initialize()` to set initial AppKit appearance.

On the `Settings` scene content:
- Add `.preferredColorScheme(appearanceMode.colorScheme)` to `SettingsView()`. This ensures the Settings window also tracks the theme (separate SwiftUI scene).

Add a `private func applyAppKitAppearance(_ mode: AppearanceMode)` method:
- `.system` -> `NSApp.appearance = nil` (tracks OS in real time)
- `.light` -> `NSApp.appearance = NSAppearance(named: .aqua)`
- `.dark` -> `NSApp.appearance = NSAppearance(named: .darkAqua)`

Important: Do NOT wrap the onChange in withAnimation — the preferredColorScheme change itself triggers SwiftUI's color cross-fade automatically. The existing `.animation(.easeInOut(duration: 0.3), value: ...)` on ContentView handles animation of color transitions. The NSApp.appearance change is imperative and does not animate (AppKit surfaces snap, which is correct native macOS behavior).

**2. Create `AppearanceTab` private struct:**

```swift
private struct AppearanceTab: View {
    @AppStorage("appearanceMode") private var appearanceMode: AppearanceMode = .system

    var body: some View {
        Form {
            Section("Theme") {
                Picker("Appearance", selection: $appearanceMode) {
                    ForEach(AppearanceMode.allCases, id: \.self) { mode in
                        Image(systemName: mode.symbolName)
                            .accessibilityLabel(mode.accessibilityLabel)
                            .tag(mode)
                    }
                }
                .pickerStyle(.segmented)

                // Color preview swatches
                HStack(spacing: Spacing.xs) {
                    ColorSwatch(color: Ayu.bgPrimary, label: "Background")
                    ColorSwatch(color: Ayu.bgSecondary, label: "Surface")
                    ColorSwatch(color: Ayu.fgPrimary, label: "Text")
                    ColorSwatch(color: Ayu.accent, label: "Accent")
                }
                .padding(.top, Spacing.xxs)
            }

            Section("Sidebar Style") {
                Text("Coming in a future update")
                    .foregroundStyle(Ayu.fgMuted)
                    .font(AppFont.caption)
            }
        }
        .formStyle(.grouped)
        .padding()
    }
}
```

Per user decisions (all locked):
- Segmented picker with SF Symbol icons only, no text labels
- Three segments: moon.fill / circle.lefthalf.filled / sun.max.fill
- Color swatch row beneath picker showing bg + fg + accent colors
- Sidebar Style placeholder section for Phase 4

**3. Create `ColorSwatch` private struct:**

```swift
private struct ColorSwatch: View {
    let color: Color
    let label: String

    var body: some View {
        RoundedRectangle(cornerRadius: Radius.xs)
            .fill(color)
            .frame(width: 32, height: 32)
            .overlay(
                RoundedRectangle(cornerRadius: Radius.xs)
                    .strokeBorder(Ayu.fgMuted.opacity(0.3), lineWidth: 1)
            )
            .accessibilityLabel(label)
    }
}
```

The swatches update live because `Ayu.*` colors use `Color.adaptive()` which resolves via NSColor appearance callbacks — when the color scheme changes, the swatches automatically show the correct theme colors.

**Important notes:**
- The `@AppStorage("appearanceMode")` key MUST match exactly the key used in GenreUpdaterApp.swift. Both read/write the same UserDefaults key, so changing the picker immediately updates the app appearance.
- Do NOT add `@Environment(AppDependencies.self)` to AppearanceTab — it has no dependency on AppDependencies.
- The Appearance tab does NOT need an `.animation()` modifier — SwiftUI handles the color cross-fade via the preferredColorScheme mechanism in GenreUpdaterApp.
