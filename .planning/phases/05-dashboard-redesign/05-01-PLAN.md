  <name>Task 2: Extend DashboardViewModel with loading states, cached-first pattern, and trends</name>
  <files>App/ViewModels/DashboardViewModel.swift</files>
  <action>
    Rewrite `DashboardViewModel.swift` to support the two-phase cached-first loading pattern:

    **Add DashboardLoadingState enum** (above the class, in the same file):
    ```
    enum DashboardLoadingState: Equatable, Sendable {
        case shimmer                     // First launch, no cache
        case cached(lastUpdated: Date)   // Showing cached data, live scan pending
        case updating                    // Live scan in progress
        case live                        // Showing live data
        case error(String)               // Scan failed
        case permissionDenied            // MusicKit access denied
        case emptyLibrary                // 0 tracks in Music.app
    }
    ```

    **Add DashboardMetrics struct** (Sendable, in the same file):
    Holds all computed metric values: `totalTracks`, `tracksWithGenre`, `tracksWithYear`, `tracksWithBoth` (consistency), `tracksNeedingGenre`, `tracksNeedingYear`, `recentlyAdded`, `genreCoverage`, `yearCoverage`, `consistencyCoverage`. Plus a static `empty` case.

    **Add TrendDirection enum** (move from MetricCard.swift to here since it's ViewModel concern):
    Keep as-is (up/down/flat with icon and tint).

    **Rewrite the class** (`@Observable @MainActor final class DashboardViewModel`):

    Published properties:
    - `private(set) var loadingState: DashboardLoadingState = .shimmer`
    - `private(set) var metrics: DashboardMetrics = .empty`
    - `private(set) var previousMetrics: DashboardMetrics?` (for trend calculation)

    Methods:
    - `func loadCachedMetrics(from snapshot: PersistedMetricsSnapshot?)` — if snapshot nil, set `loadingState = .shimmer`; otherwise build `DashboardMetrics` from snapshot, set `previousMetrics` from snapshot's `previous*` fields, set `loadingState = .cached(lastUpdated: snapshot.timestamp)`.
    - `func refreshFromLive(tracks: [Track])` — compute `DashboardMetrics` from tracks array in single pass (same O(n) logic as current `refresh`). Save current metrics as `previousMetrics` if current has data. Update `loadingState` to `.live` (or `.emptyLibrary` if tracks.count == 0). Remove `topGenres` computation (moved to Reports per CONTEXT.md decision).
    - `func setPermissionDenied()` — sets `loadingState = .permissionDenied`.
    - `func setError(_ message: String)` — sets `loadingState = .error(message)`.

    Computed trend properties:
    - `var genreTrend: TrendDirection?` — compare `metrics.tracksNeedingGenre` to `previousMetrics?.tracksNeedingGenre`. Less needing = `.down` (good), more = `.up`, same = `.flat`. Return nil if no previous.
    - `var yearTrend: TrendDirection?` — same logic for year.
    - `var recentTrend: TrendDirection?` — compare recentlyAdded. More = `.up`, less = `.down`, same = `.flat`.
    - `var genreTrendDelta: Int?` — `metrics.tracksNeedingGenre - (previousMetrics?.tracksNeedingGenre ?? 0)` if previous exists, nil otherwise.
    - `var yearTrendDelta: Int?` — same for year.
    - `var recentTrendDelta: Int?` — same for recently added.

    Consistency metric in the single-pass computation:
    - Track has genre if `track.genre != nil && !track.genre!.isEmpty` (empty string treated as nil per pitfall).
    - Track has year if `track.year != nil`.
    - `tracksWithBoth` = tracks where BOTH conditions are true.

    Import `import Services` for PersistedMetricsSnapshot access. Keep `import Core` and `import Foundation`.
    Remove `import Observation` (not needed with @Observable macro).
    Remove `topGenres` and `uniqueGenres` / `uniqueArtists` properties — they are not used on the redesigned Dashboard.
    Remove `isLoading` bool — replaced by `loadingState` enum.

    Note: TrendDirection for "Need Genre" and "Need Year" is inverted from typical — fewer tracks needing attention is GOOD (down arrow = green), more needing = BAD (up arrow = red). For "Recently Added", more is neutral/positive.
