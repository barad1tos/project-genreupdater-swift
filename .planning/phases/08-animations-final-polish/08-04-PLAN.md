**ReportsCharts.swift** -- Chart bar animation (grow from zero):

1. Add `@State private var chartAnimated = false` for each chart (genre distribution and year distribution).
2. Add `@Environment(\.accessibilityReduceMotion) private var reduceMotion`
3. Add `@Environment(\.motionScale) private var motionScale`

For the genre distribution horizontal bar chart:
```swift
BarMark(
    x: .value("Count", chartAnimated ? item.count : 0),
    y: .value("Genre", item.genre)
)
```

For the year distribution chart:
```swift
BarMark(
    x: .value("Year", item.year),
    y: .value("Count", chartAnimated ? item.count : 0)
)
```

On each chart's `.onAppear`:
```swift
.onAppear {
    guard !chartAnimated else { return }
    if reduceMotion {
        chartAnimated = true
        return
    }
    withAnimation(Motion.scaled(.springOrganic, by: motionScale)) {
        chartAnimated = true
    }
}
```

The springOrganic token (response 0.5, damping 0.7) gives bars an organic grow-in with slight overshoot. Per RESEARCH: "Swift Charts animate marks when their data values change. Setting value from 0 to actual with withAnimation causes bars to grow."

**Chart hover tooltip:**

For each chart, add a `@State private var hoveredItem: String? = nil` (genre name or year string).

Add `.chartOverlay { proxy in ... }` with a GeometryReader containing a transparent Rectangle with `.contentShape(.rect)` and `.onContinuousHover`:

```swift
.chartOverlay { proxy in
    GeometryReader { geometry in
        Rectangle()
            .fill(.clear)
            .contentShape(.rect)
            .onContinuousHover { phase in
                switch phase {
                case .active(let location):
                    // For horizontal bars (genre): map Y position to genre
                    if let genre: String = proxy.value(atY: location.y) {
                        hoveredItem = genre
                    }
                case .ended:
                    hoveredItem = nil
                }
            }
    }
}
```

For genre chart (horizontal bars): use `proxy.value(atY:)` since genres are on Y axis.
For year chart (vertical bars): use `proxy.value(atX:)` since years are on X axis.

Dim non-hovered bars by applying `.opacity(hoveredItem == nil || hoveredItem == item.genre ? 1.0 : 0.3)` to each BarMark.

Add a tooltip overlay using `.annotation(position:)` on the BarMark when the item matches hoveredItem. Style the tooltip capsule with `Ayu.bgSecondary` background, `Ayu.fgPrimary` text, and `Shadow.medium` -- these are the actual DesignTokens connections for tooltip styling:
```swift
BarMark(...)
    .annotation(position: .trailing, spacing: 4) {
        if hoveredItem == item.genre {
            Text("\(item.count)")
                .font(AppFont.caption)
                .foregroundStyle(Ayu.fgPrimary)
                .padding(.horizontal, Spacing.xs)
                .padding(.vertical, Spacing.xxs)
                .background(Ayu.bgSecondary, in: .capsule)
                .ayuShadow(Shadow.medium)
        }
    }
```

For year chart, use `.annotation(position: .top)` since bars are vertical.

**ReportsChangeLog.swift** -- Smooth list insertion/removal:

Add `.transition(.opacity.combined(with: .move(edge: .top)))` on each change log entry row within the ForEach. This creates smooth insertion (fade + slide from top) and removal (fade + slide to top) animations.

Ensure the ForEach has proper item identity (using entry ID) so SwiftUI can track insertions/removals.

Per CONTEXT.md: "Change log new entries: instant -- data over effects." This means NEW entries appearing during a live session should appear instantly. The `.transition` should only animate on explicit user actions (like undo removing an entry). If the change log receives streaming entries, do NOT wrap those updates in `withAnimation`. The transition modifier itself is inert unless the state change is wrapped in `withAnimation` -- so adding the modifier is safe. Undo operations (which remove entries) should be wrapped in `withAnimation` to trigger the smooth removal.
