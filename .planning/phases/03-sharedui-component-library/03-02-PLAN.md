**ArtistListRow** (`Components/ArtistListRow.swift`):
- `public struct ArtistListRow: View` with parameters:
  - `name: String` -- artist name
  - `albumCount: Int` -- number of albums
  - `trackCount: Int` -- number of tracks
  - `isSelected: Bool` -- persistent selection state (controlled by parent)
- `@State private var isHovered = false`
- `@State private var isPressed = false`
- Layout: `HStack` with:
  1. Leading accent bar: `RoundedRectangle(cornerRadius: 2).fill(Ayu.accent).frame(width: 3)` -- opacity 1 when `isHovered || isSelected`, opacity 0 otherwise, animated with `Motion.curveFast`
  2. Artist name: `Text(name)` with `AppFont.body`, `Ayu.fgPrimary`, left-aligned, `.lineLimit(1)`
  3. Spacer
  4. Album count badge: `Text("\(albumCount)a")` with `.system(.caption, design: .monospaced)` font, `Ayu.fgSecondary`, inside a small pill with `Ayu.bgTertiary` background and `Radius.xs` corner radius, horizontal padding `Spacing.xxs + 2` (6pt), vertical padding 2pt
  5. Track count badge: `Text("\(trackCount)t")` -- same style as album badge, with `Spacing.xxs` gap between badges
- Row height: `.frame(height: 44)`
- Background: `RoundedRectangle(cornerRadius: Radius.xs)` filled with:
  - `isSelected ? Ayu.accent.opacity(0.1)` (persistent selection)
  - `: isHovered ? Ayu.bgTertiary.opacity(0.5)` (hover highlight)
  - `: Color.clear` (default)
- NO dividers between rows -- spacing only (consumer handles spacing)
- `.contentShape(.rect)` -- CRITICAL for macOS 15 scroll regression
- `.onHover { hovering in withAnimation(Motion.curveFast) { isHovered = hovering } }`
- Press: `.scaleEffect(isPressed ? 0.98 : 1.0)` with `.animation(Motion.curveFast, value: isPressed)`
- Use `.simultaneousGesture(DragGesture(minimumDistance: 0).onChanged { _ in isPressed = true }.onEnded { _ in isPressed = false })` for press detection. NOTE: Do not call any action in the gesture -- the parent handles tap/click via the List selection binding or .onTapGesture.
- `.focusable()` for keyboard focus
- Accessibility: `.accessibilityElement(children: .combine)` to group name + badges; `.accessibilityLabel("\(name), \(albumCount) albums, \(trackCount) tracks")`; `.accessibilityAddTraits(isSelected ? .isSelected : [])`
- `public init(name:albumCount:trackCount:isSelected:)` -- all public
- `#Preview` block: a VStack showing 5 artists (e.g., "Radiohead", "Daft Punk", "Bjork", "Massive Attack", "Portishead") with varying counts, one marked as selected, demonstrating normal/hover/selected states

  <files>Packages/SharedUI/Sources/SharedUI/Components/SectionIndexBar.swift</files>
  <action>
Create `Packages/SharedUI/Sources/SharedUI/Components/SectionIndexBar.swift`:

- `public struct SectionIndexBar: View` with parameters:
  - `letters: [String]` -- only letters with content (smart mode -- consumer computes which letters have artists). Generic String array, not hardcoded A-Z. Supports "#" for non-alphabetic sections.
  - `onLetterSelected: (String) -> Void` -- callback when user taps or drags to a letter. Consumer uses this with ScrollViewReader to scroll to the section.
- `@State private var activeLetter: String?` -- the letter currently under the user's finger during drag
- `@State private var isDragging = false`
- Layout: `VStack(spacing: 2)` containing `ForEach(letters, id: \.self)` of letter views
- Each letter: `Text(letter)` with `.font(.system(size: 11, weight: .semibold))`, `Ayu.fgSecondary` foreground (or `Ayu.accent` if it's the activeLetter during drag), `.frame(width: 16, height: 14)` for uniform sizing
- When `isDragging`, show a subtle background on the bar: `Ayu.bgTertiary.opacity(0.5)` with `Radius.xs` corner radius
- Drag gesture: `.gesture(DragGesture(minimumDistance: 0).onChanged { value in ... }.onEnded { _ in ... })`
  - In `.onChanged`: set `isDragging = true`, calculate letter index from `value.location.y` divided by per-letter height (14 + 2 spacing = 16pt per letter). Clamp to valid range. If the calculated letter differs from `activeLetter`, update `activeLetter` and call `onLetterSelected(letters[index])`
  - In `.onEnded`: set `isDragging = false`, clear `activeLetter`
- Place SectionIndexBar OUTSIDE/overlay on the List, NOT inside it -- this prevents coordinate space issues (document this in a code comment for Phase 6 consumer guidance)
- `.frame(width: 20)` for the overall bar width (16pt letters + 2pt padding each side)
- `.padding(.vertical, Spacing.xxs)` for top/bottom breathing room
- Accessibility: `.accessibilityLabel("Section index")`, `.accessibilityHint("Drag to jump to a section")`
- `public init(letters:onLetterSelected:)` -- both public
- Add `#Preview` block showing a SectionIndexBar with letters ["A", "B", "C", "D", "E", "F", "G", "H", "I", "J", "K", "L", "M", "N", "O", "P", "R", "S", "T", "W"] (smart mode -- skipping Q, U, V, X, Y, Z) alongside a placeholder List area. Include a comment showing the Phase 6 consumption pattern:
