   - Body renders the appropriate shape filled with `Ayu.bgTertiary` (opaque, not glass-backed per research recommendation) and applies `.shimmering()`

   - Mark all public types and inits as `public`

**FilterChip** (`Components/FilterChip.swift`):
- `public struct FilterChip: View` with parameters:
  - `label: String` -- chip text
  - `isActive: Bool` -- controls active/inactive appearance
  - `isDismissable: Bool = false` -- shows xmark.circle when active + dismissable
  - `onTap: () -> Void` -- toggle callback
  - `onDismiss: (() -> Void)? = nil` -- dismiss callback
- `@State private var isHovered = false`
- `@State private var isPressed = false`
- Active state: `Ayu.accent` background fill + `.white` foreground text
- Inactive state: border only using `Ayu.fgMuted.opacity(0.3)` strokeBorder (lineWidth 1) + `Ayu.fgPrimary` foreground text
- Shape: `RoundedRectangle(cornerRadius: Radius.sm)` -- squared tag style, NOT capsule
- When `isDismissable && isActive`: show `Image(systemName: "xmark.circle")` in `.caption2` font after the label text, with `.onTapGesture { onDismiss?() }`
- Horizontal padding: `Spacing.xs`, vertical padding: `Spacing.xxs`
- Font: `AppFont.caption`
- `.contentShape(.rect)` for hit testing
- Press state: `.scaleEffect(isPressed ? 0.98 : 1.0)` with `Motion.curveFast` animation on isPressed
- `.onHover { isHovered = $0 }` (hover state reserved for future visual feedback -- wire the state but no visual change beyond press/active for now)
- Toggle animation: `.animation(Motion.curveFast, value: isActive)` for the color cross-fade (~0.2s)
- Wrap the whole thing in a `Button` action for press detection (using a custom `ButtonStyle` that sets isPressed), OR use `.pressAction(onPress: { isPressed = true }, onRelease: { isPressed = false })` via a custom gesture. Recommended: use a simple `.simultaneousGesture(DragGesture(minimumDistance: 0).onChanged { _ in isPressed = true }.onEnded { _ in isPressed = false; onTap() })` pattern. The important thing is the 0.98x scale on press.
- `.focusable()` for keyboard focus support
- `.accessibilityLabel(label)` and `.accessibilityAddTraits(isActive ? .isSelected : [])` for VoiceOver
- Add `#Preview` with examples: active chip, inactive chip, dismissable active chip, a row of mixed chips

**StatCard** (`Components/StatCard.swift`):
- `public struct StatCard: View` with parameters:
  - `label: String` -- small label text above the number
  - `value: String` -- the big number (String to allow formatting like "38,085")
  - `progress: Double` -- 0.0-1.0 for the mini progress bar
  - `onTap: (() -> Void)? = nil` -- optional tap callback
- `@State private var isHovered = false`
- `@State private var isPressed = false`
- `@Environment(\.accessibilityReduceMotion) private var reduceMotion`
- Background: `Ayu.bgSecondary` filled `RoundedRectangle(cornerRadius: Radius.md)`
- Shadow: `.ayuShadow(isHovered ? Shadow.elevated : Shadow.subtle)` -- elevation changes on hover
- Border: when hovered, overlay a `RoundedRectangle(cornerRadius: Radius.md).strokeBorder(Ayu.accent, lineWidth: 1.5)` that fades in simultaneously with the shadow change
- Layout (VStack, alignment: .leading, spacing: Spacing.xs):
  1. Label: `AppFont.caption`, `Ayu.fgSecondary`
  2. Value: `AppFont.metric`, `Ayu.fgPrimary`
  3. Mini progress bar: a `GeometryReader` or fixed-width approach -- full-width track in `Ayu.bgTertiary` (height 4pt, cornerRadius 2pt), filled portion in `Ayu.accent` with `.animation(Motion.curveDefault, value: progress)` for smooth width transition
- Padding: `Spacing.md` all around
- `.contentShape(.rect)` for hit testing
- `.onHover { hovering in withAnimation(Motion.curveFast) { isHovered = hovering } }`
- Press state: `.scaleEffect(isPressed ? 0.98 : 1.0)` with `Motion.curveFast` animation
- Use `.simultaneousGesture(DragGesture(minimumDistance: 0)...)` for press detection, call `onTap?()` on release
- `.focusable()` for keyboard focus
- `.accessibilityElement(children: .combine)` to group label+value for VoiceOver
- Add `#Preview` with a 2-column grid of 4 StatCards showing different values and progress levels, in both light and dark color schemes
