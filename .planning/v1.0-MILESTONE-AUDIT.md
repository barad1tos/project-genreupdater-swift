## Integration Issues

### Issue 1 (HIGH): Shimmer dismissed prematurely on first launch

**Files:** `DashboardView.swift:45-47`
**Requirements:** DASH-02, DASH-03

`.task(id: tracks.count)` fires at initial render with `tracks = []` (count=0) while `isLoading=true`. `refreshFromLive(tracks: [])` sets `loadingState = .emptyLibrary`, replacing shimmer before MusicKit data arrives. When tracks arrive, state jumps to `.live` — a visible flash.

**Fix:** Guard on `isLoadingTracks` inside the task body:
```swift
.task(id: tracks.count) {
    guard !isLoadingTracks || !tracks.isEmpty else { return }
    viewModel.refreshFromLive(tracks: tracks)
}
```

### Issue 2 (MEDIUM): ShimmerPlaceholder receives CGFloat.infinity

**Files:** `DashboardView.swift:215, 217`
**Requirement:** DASH-03

`ShimmerPlaceholder(shape: .rectangle(width: .infinity, height: 44))` passes `CGFloat.infinity` as fixed `.frame(width:)`. The outer `.frame(maxWidth: .infinity)` does not override the inner fixed frame.

**Fix:** Use `shape: .card` or modify `ShimmerShape.rectangle` to use `maxWidth` internally.
